# SmartBuy AI｜後台 Dashboard 視覺樣式參考

> 本文件把使用者提供的後台參考畫面轉成 SmartBuy AI 可執行規範。只參考資訊架構與視覺語言，不複製原品牌、Logo、文字或資料。

## 1. 視覺方向

- 深墨綠固定側欄
- 白色頂部工具列
- 淺灰／暖灰內容背景
- 白色卡片、低干擾陰影、淡色邊框
- 綠色：正常、成長、主要操作
- 黃橘色：注意、等待
- 紅色：錯誤、高風險、失敗
- 藍色：資訊、排程、中性系統狀態
- 顏色不可作為唯一狀態判斷
- 禁止複製參考畫面的品牌與假資料

## 2. Desktop 骨架

### Sidebar

- 固定左側，寬度 `232px～248px`
- 頂端顯示 `SmartBuy AI 管理後台`
- 導覽使用圖示 + 文字
- Active 項目使用亮綠底或左側強調條
- 群組以間距或分隔線區隔
- 底部可顯示版本、環境與登入角色

### Topbar

- 高度 `64px～72px`
- 左側：頁面標題、麵包屑
- 右側：系統狀態、通知、角色、使用者選單
- 主要導覽仍由 Sidebar 負責

### Main Content

- 最大寬度 `1440px`
- 內距 `24px～32px`
- 資訊順序：頁首操作 → KPI → 圖表／健康狀態 → 表格／任務清單

## 3. KPI 卡片

- Desktop 四欄、Tablet 兩欄、Mobile 一欄
- 顯示名稱、主數值、補充說明、圖示、資料來源與時間
- API 失敗不得顯示為 `0`
- 未接 API 顯示 `—` 與「尚未接入」
- 圓角 `10px～14px`
- 主數值 `28px～36px`
- 狀態需同時使用文字 Badge

## 4. 圖表區

- 主趨勢圖可使用折線圖或面積圖
- 健康狀態可使用甜甜圈圖、橫條圖或狀態列表
- 圖表旁必須提供文字摘要
- Chart.js 更新與 unmount 時必須 destroy
- Desktop 建議主圖 `2/3` + 狀態卡 `1/3`
- Mobile 全部單欄並減少標籤

## 5. 表格與任務清單

- 表格支援排序、篩選、分頁
- 可點擊列支援 Enter／Space
- Desktop 顯示完整欄位
- Tablet 隱藏次要欄位
- Mobile 改卡片清單
- 任務清單顯示任務名稱、狀態、時間與操作
- 沒有正式任務 API 時不得編造紀錄或成功率

## 6. 三尺寸

### Mobile `0～767px`

- Sidebar 改 Drawer
- KPI、圖表、表格全部單欄
- 篩選器使用 Drawer／Bottom Sheet
- 左右內距 `16px`
- 禁止全頁水平捲軸

### Tablet `768～1199px`

- Sidebar 縮為 `72px` 圖示列
- KPI 兩欄
- 圖表依可讀性一欄或兩欄
- 內容內距 `24px`

### Desktop `1200px+`

- 完整固定 Sidebar
- KPI 四欄
- 圖表兩欄
- 完整表格

## 7. SmartBuy AI 轉譯

- 使用農業綠、深綠、米白、淺灰
- 導覽圖示對應行情、農產品、AI、天氣、節氣、互助網、會員、排程、設定
- 不出現與本系統無關的倉庫、訂單或物流假資料
- 所有 KPI 必須屬於正式 API、明確 Demo 或 Unavailable

## 8. 驗收

後台 UI PR 必須提供：

- 與本文件的對照
- 三尺寸行為
- Design Token
- 正式 API／Demo／Unavailable 清單
- Loading／Empty／Error／Partial Error／Stale
- 360、390、768、834、1200、1440 六尺寸驗證
