# SmartBuy AI｜消費者首頁視覺與資訊架構參考

> 參考來源：農業部食農教育資訊整合平臺「臺灣農產地圖」  
> 來源頁面：https://fae.moa.gov.tw/map/county_agri.php  
> 本文件只參考資訊架構、互動方式與地方農產的呈現邏輯，不複製原網站品牌、Logo、插圖、版面素材或程式碼。

---

## 1. 首頁定位

SmartBuy AI 首頁不是政府資料入口，也不是後台數據牆，而是給一般消費者使用的「今天買什麼」採買首頁。

首頁需要同時完成兩件事：

1. **快速決策**：立即查菜價、看便宜／正常／偏貴、取得採買建議。
2. **農產探索**：了解當月盛產、在地特色與農產來源地。

正式首頁架構採：

```text
快速採買決策
→ 當月與在地探索
→ 價格／節氣／產地資訊
→ 收藏、提醒、新知與互助服務
```

---

## 2. 參考頁可吸收的核心設計

參考頁的重要資訊架構包括：

- 「在地特色」
- 「本月尚青」
- 「農產在哪」
- 臺灣縣市互動探索
- 縣市與月份篩選
- 各縣市特色農產品
- 盛產月份與品項
- 生產縣市排名、栽培面積與年產量
- 清楚標示資料來源

SmartBuy AI 應吸收上述探索邏輯，但要改寫為更簡單、可採買、手機優先的消費者介面。

## 3. SmartBuy AI 首頁正式資訊順序

### 3.1 第一屏｜今天買什麼

第一屏必須保留目前核心：

- 標題：`今天買什麼？`
- 品項搜尋
- 市場選擇
- 目前節氣
- 主要 CTA：`查今天菜價`
- 一句白話說明

第一屏禁止：

- 放完整臺灣地圖
- 放大量統計圖表
- 一次顯示全部縣市
- 顯示專業模型名詞
- 用官方入口網站式多層選單壓縮主內容

### 3.2 第二區｜三種農產探索入口

建立三個清楚入口：

```text
在地特色
本月尚青
農產在哪
```

#### 在地特色

回答：

- 我的縣市有什麼代表農產？
- 這些品項現在價格如何？
- 哪些品項值得進一步查看？

顯示內容：

- 縣市選擇
- 3～6 個代表品項
- 品項圖示或合法授權圖片
- 目前價格狀態
- 前往商品詳情

#### 本月尚青

回答：

- 這個月盛產什麼？
- 哪些品項當季、價格可能比較合理？
- 適合怎麼料理？

顯示內容：

- 當月月份
- 當季品項
- 目前節氣
- 價格狀態
- 料理建議
- 資料來源標籤

#### 農產在哪

回答：

- 某個品項主要來自哪些縣市？
- 哪裡是主要產區？
- 不同產區的行情或資料是否可取得？

顯示內容：

- 品項搜尋
- 臺灣縣市地圖或縣市清單
- 主要產區
- 生產資訊摘要
- 前往行情查詢

## 4. 臺灣地圖設計

### 4.1 地圖不是第一屏主角

地圖放在第二區或第三區，避免阻礙使用者快速查價。

### 4.2 Desktop

- 左側：臺灣互動地圖
- 右側：選取縣市的特色農產卡片
- Hover 顯示縣市名稱
- Click／Enter／Space 選取縣市
- 選取狀態需有文字，不只靠顏色

### 4.3 Tablet

- 地圖與內容可上下排列
- 縣市選擇另提供下拉選單
- 地圖不得過小導致難以點選

### 4.4 Mobile

Mobile 不強制使用小型 SVG 地圖作為唯一操作方式。

優先提供：

- 北部／中部／南部／東部及外島區域切換
- 縣市 Chips
- 縣市下拉選單
- 可選的簡化地圖預覽

必須保證 360px 寬度可完整操作。

## 5. 首頁完整區塊建議

```text
1. Hero：今天買什麼 + 搜尋 + 節氣
2. 今日採買建議：便宜／正常／偏貴
3. 農產探索 Tabs：在地特色／本月尚青／農產在哪
4. 臺灣產地探索：地圖或縣市清單
5. 本月盛產品項：當季品項 + 行情狀態
6. 在地故事：縣市代表農產與簡短介紹
7. 收藏與提醒摘要
8. 農產新知與互助網
9. 資料來源與更新時間
```

首頁不得因新增農產探索而移除：

- 菜價搜尋
- 今日採買建議
- 商品詳情入口
- 節氣入口
- 收藏與提醒
- 農產新知
- 互助網
- Loading／Empty／Error

## 6. 視覺語言

### 6.1 風格

- 明亮、溫暖、生活化
- 農業綠 + 米白 + 淡黃色
- 地圖色塊柔和，不使用後台深色科技風
- 卡片圓角與插圖可以更親切
- 避免政府入口網站的高密度導覽感

### 6.2 色彩用途

| 用途 | 建議 |
|---|---|
| 主要品牌與 CTA | 農業綠 |
| 當季／盛產 | 淡黃或暖橘 |
| 地圖選取 | 深綠邊框 + 淺綠底 |
| 便宜 | 綠色 + 文字 |
| 正常 | 黃褐色 + 文字 |
| 偏貴 | 紅色 + 文字 |
| 資料不足 | 灰色 + 說明 |

顏色不得作為唯一資訊辨識方式。

## 7. 資料來源原則

### 7.1 參考網站只作設計參考

禁止：

- 直接複製原網站 HTML、CSS 或 JavaScript
- 下載並重用原網站 Logo、地圖插圖或版權素材
- 未經確認直接爬取頁面作為正式資料庫
- 將參考頁內容包裝成 SmartBuy AI 自有資料

### 7.2 正式資料應優先使用

- 農業部農產品交易行情
- 農業資料開放平臺
- 已確認授權與格式的縣市盛產資料
- SmartBuy AI 現有正式行情 API
- `/api/solar-term`

### 7.3 資料狀態標示

所有首頁探索資料必須標示：

```text
Official API
Static Seed
Demo
Unavailable
```

若縣市特色或盛產資料尚未串接正式 API：

- 可建立介面骨架
- 必須標示 Static Seed 或 Unavailable
- 不得以固定假資料冒充即時官方資料


- Partial Error

禁止把資料拼接與來源判斷全部寫進 `Home.jsx`。

## 9. 三尺寸 RWD

### Mobile `0～767px`

- Hero 單欄
- 搜尋欄與市場選擇上下排列
- 三個探索入口可橫向滑動或等寬 Tabs
- 縣市以 Chips／下拉為主
- 品項卡片單欄或橫向卡片列
- 地圖不可成為唯一操作入口
- 左右內距 16px

### Tablet `768～1199px`

- Hero 可雙欄
- 探索區地圖與資訊上下或 5:7 排列
- 品項卡片兩欄
- 內容內距 24px

### Desktop `1200px+`

- Hero 雙欄
- 探索區地圖與內容左右排列
- 品項卡片 3～4 欄
- 最大寬度 1200px
- 內容內距 24～32px

## 10. Accessibility

- 地圖縣市可用鍵盤選取
- 地圖同時提供文字縣市清單
- Tabs 使用正確 tab semantics
- 圖片有替代文字
- Icon button 有 `aria-label`
- Focus 樣式清楚
- 觸控區至少 44 × 44px
- 地圖顏色搭配文字與邊框
- Reduced Motion 下停用非必要動畫

---

## 11. 驗收條件

- 首屏仍能在 5 秒內理解「今天買什麼」
- 搜尋與今日採買建議不因改版消失
- 三個探索入口可操作
- Mobile 可不用地圖完成縣市查詢
- 地圖與縣市清單狀態同步
- 當月品項標示資料來源與更新時間
- 正式行情 API 失敗時顯示 Partial Error
- 不把失敗資料顯示為正常或 0
- 360、390、768、834、1200、1440 六尺寸無主要破版
- 不直接複製參考網站品牌與素材

---

## 12. 開發切分建議

首頁探索功能應獨立成一個 PR，不要混入 AI 預測或後台管理 PR。

建議順序：

1. 建立首頁探索骨架與 Tabs
2. 建立縣市選擇與 Mobile fallback
3. 接入節氣與現有行情 API
4. 接入正式縣市／月份農產資料
5. 最後加入臺灣地圖與合法素材

第一版若沒有正式縣市農產 API，應先完成介面、資料 contract 與 Unavailable 狀態，不得先填滿假資料。
